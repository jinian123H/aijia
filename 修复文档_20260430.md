## 问题一：首页推荐 JSON 解析崩溃

### 日志
```
JsonSyntaxException: Expected BEGIN_ARRAY but was BEGIN_OBJECT at line 1 column 2 path $.data.content
  VideoRepository.kt:618
```

### 根因
`VideoRepository.getHomeRecommend()` 使用 `HomeRecommendVersionedData` 解析 API 响应，该 data class 声明 `content: List<HomeSection>?`，期望 `content` 是 JSON 数组。

但后端 `api/v1/vod/index` 实际返回的 `content` 是 JSON **对象**：
```json
{
  "content": {
    "banners": [...],
    "sections": [...]
  }
}
```

### 涉及文件

| 文件 | 行号 | 说明 |
|------|------|------|
| `data/model/VersionedData.kt` | L64-68 | `HomeRecommendVersionedData` 类型错误 |
| `data/repository/VideoRepository.kt` | L608-660 | `getHomeRecommend()` 使用错误类型 |

### 修复方案

**方案：改用 `HomeIndexVersionedData`（已有正确模型）**

`VersionedData.kt` 中已存在 `HomeIndexVersionedData`，其 `content` 类型为 `HomeIndexResponse?`（含 `banners` + `sections` 两个字段）。

修改 `VideoRepository.kt:618`：
```kotlin
// 旧（错误）
val type = object : TypeToken<ApiResponse<HomeRecommendVersionedData>>() {}.type
val apiResponse: ApiResponse<HomeRecommendVersionedData> = gson.fromJson(json, type)

// 新（正确）
val type = object : TypeToken<ApiResponse<HomeIndexVersionedData>>() {}.type
val apiResponse: ApiResponse<HomeIndexVersionedData> = gson.fromJson(json, type)
```

修改 `VideoRepository.kt:628`（返回值提取）：
```kotlin
// 旧
Result.success(versionedData.content)

// 新
Result.success(versionedData.content?.sections)
```

同步修改 `VideoRepository.kt:633`（缓存回退读取）不需要改动，因为缓存存储的已经是 `List<HomeSection>` 的 JSON。

同步修改缓存写入逻辑 `VideoRepository.kt:625`：
```kotlin
// 旧
.putString(KEY_CACHED_DATA, gson.toJson(versionedData.content))

// 新
.putString(KEY_CACHED_DATA, gson.toJson(versionedData.content?.sections))
```

## 问题二：Banner 接口 404

### 日志
```
HTTP 404: {"code":404,"msg":"接口不存在"}
  GET /api/v1/app/banner
```

### 根因
`ApiService.kt:L296-299` 定义的 `getBanner()` 调用 `GET api/v1/app/banner`，但 Go 后端没有此路由。

### 涉及文件

| 文件 | 行号 | 说明 |
|------|------|------|
| `data/remote/ApiService.kt` | L296-299 | Banner API 定义 |
| `data/repository/VideoRepository.kt` | L666-700 | `getBanner()` 实现 |

### 修复方案

**方案：从首页数据中提取 Banner（类似 Rust 桌面端 `get_banners()` 模式）**

Banner 数据已内嵌在 `api/v1/vod/index` 响应的 `data.content.banners` 中。修改 `VideoRepository.getBanner()` 改为从首页数据提取，而非独立接口。

```kotlin
suspend fun getBanner(): Result<List<BannerItem>?> {
    return try {
        // 直接依赖 getHomeIndex 接口（不经过 getHomeRecommend），获取完整首页数据
        val versionId = bannerPrefs.getString(KEY_BANNER_VERSION_ID, "") ?: ""
        val response = apiService.getHomeIndex(versionId)

        if (response.isSuccessful) {
            val json = response.body()?.string() ?: return Result.failure(Exception("Empty response"))
            val type = object : TypeToken<ApiResponse<HomeIndexVersionedData>>() {}.type
            val apiResponse: ApiResponse<HomeIndexVersionedData> = gson.fromJson(json, type)

            if (apiResponse.code == 200) {
                val versionedData = apiResponse.data
                val banners = versionedData.content?.banners?.map { video ->
                    BannerItem(
                        id = video.id.toIntOrNull() ?: 0,
                        title = video.name,
                        cover = video.picSlide,
                        vodId = video.id.toIntOrNull() ?: 0
                    )
                }

                if (versionedData.dataChanged && banners != null) {
                    bannerPrefs.edit()
                        .putString(KEY_BANNER_VERSION_ID, versionedData.versionId)
                        .putString(KEY_BANNER_DATA, gson.toJson(banners))
                        .apply()
                    Result.success(banners)
                } else {
                    val cachedJson = bannerPrefs.getString(KEY_BANNER_DATA, null)
                    if (cachedJson != null) {
                        val listType = object : TypeToken<List<BannerItem>>() {}.type
                        val cachedData: List<BannerItem> = gson.fromJson(cachedJson, listType)
                        Result.success(cachedData)
                    } else {
                        if (versionId.isNotEmpty()) {
                            bannerPrefs.edit().putString(KEY_BANNER_VERSION_ID, "").apply()
                            return getBanner()
                        }
                        Result.success(emptyList())
                    }
                }
            } else {
                Result.failure(Exception(apiResponse.msg))
            }
        } else {
            Result.failure(Exception("Network error: ${response.code()}"))
        }
    } catch (e: Exception) {
        android.util.Log.e("VideoRepository", "getBanner异常", e)
        val cachedJson = bannerPrefs.getString(KEY_BANNER_DATA, null)
        if (cachedJson != null) {
            val listType = object : TypeToken<List<BannerItem>>() {}.type
            val cachedData: List<BannerItem> = gson.fromJson(cachedJson, listType)
            Result.success(cachedData)
        } else {
            Result.failure(NetworkErrorHandler.formatError(e))
        }
    }
}
```

## 问题三：强制更新检查接口 404

### 日志
```
HTTP 404: {"code":404,"msg":"接口不存在"}
  POST /api/v1/app/version/check
```

### 根因
`ApiService.kt:L266` 定义的 `checkForceUpdate()` 调用 `POST /api/v1/app/version/check`，Go 后端无此路由。

### 涉及文件

| 文件 | 行号 | 说明 |
|------|------|------|
| `data/remote/ApiService.kt` | L266-269 | 强制更新 API 定义 |
| `data/repository/VersionRepository.kt` | L69-100 | `checkForceUpdate()` 实现 |
| `ui/update/UpdateViewModel.kt` | L56-109 | 调用方 |

### 修复方案

**优先方案：Go 后端添加 `api/v1/app/version/check` 路由**

Go 后端已存在 `api/v1/app/check-version` 用于普通版本检查，需新增 `api/v1/app/version/check` 路由实现强制更新逻辑：

```go
// Go 后端伪代码 — 新增路由
router.POST("/api/v1/app/version/check", func(c *gin.Context) {
    var req CheckForceUpdateRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        c.JSON(400, gin.H{"code": 400, "msg": "参数错误"})
        return
    }
    // 从 redis/DB 读取强制更新配置，比较 version_code
    // 返回 CheckForceUpdateResponse { need_update, force_update, content, update_version_id }
})
```

**备选方案：客户端降级**  
如果暂时无法修改后端，让 `UpdateViewModel.checkUpdate()` 在 `checkForceUpdate` 失败时静默跳过强制更新弹窗，不影响其他功能。
